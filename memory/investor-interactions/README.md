# Investor Interactions

Per-investor learning loop similar to playing group: capture → analyze → extract lessons → apply to next interaction.

## Structure

```
memory/investor-interactions/
├── README.md (this file)
├── message-scores.jsonl (all scored messages)
├── {investor-name}.md (per-investor profiles)
```

## Profile Template

Each investor profile includes:

### Basic Info
- Name, firm, role
- Phone/email
- Fund thesis
- Portfolio focus
- Ticket size / stage

### Conversation History
- Chronological log of all interactions
- What we discussed
- Their objections / questions
- What resonated / what didn't

### Lessons Learned
- What positioning works with them
- What analogies landed
- Red flags / things to avoid
- Follow-up items

### Strategy
- Next steps
- Key messages to emphasize
- References to prepare
- Competitive positioning

## Scoring System

Every investor message gets scored (0-10 each, total 50):

- 💡 **Intelligence-First**: Leads with trained AI, not platform
- 🎯 **Thesis Alignment**: Ties to investor's specific focus
- 🔒 **Moat Clarity**: Explains why it's defensible
- 📊 **Evidence-Backed**: Uses real metrics (742 teaching, 100+ attacks)
- 🎨 **Clarity**: Clear analogies and positioning

**Quality levels:**
- 45-50: EXCELLENT
- 35-44: GOOD
- 25-34: ACCEPTABLE
- <25: NEEDS_WORK

## ONE MESSAGE RULE

Never send multiple messages in a row to investors. Compose ONE complete, well-thought-out message.

## Critical Rules

1. **LEAD WITH INTELLIGENCE** - Never lead with "platform" or "infrastructure"
2. **TIE TO THESIS** - Research their fund before responding
3. **USE EVIDENCE** - Real numbers (742 teaching interactions, 100+ security attacks)
4. **CLEAR ANALOGIES** - Slack/Twilio, VS Code/AWS, not abstract concepts
5. **ONE MESSAGE** - No immediate follow-ups or self-corrections
