## üî¥ FAILURE PATTERNS TO AVOID

### ‚ùå Pattern #1: Multiple Messages
**Symptom:** Send pitch ‚Üí realize I forgot something ‚Üí send follow-up ‚Üí send correction  
**Fix:** Use checklist BEFORE first send. One complete message.

### ‚ùå Pattern #2: Infrastructure-First
**Symptom:** Lead with "We built a platform that..." or "Model-agnostic infrastructure"  
**Fix:** ALWAYS lead with accumulated intelligence and learning loops.

### ‚ùå Pattern #3: Competitor Claims Without Research
**Symptom:** "They're just an end-user desktop app" when they launched enterprise 2 days ago  
**Fix:** Web search competitor + "last 7 days" BEFORE any comparison.

### ‚ùå Pattern #4: Missing Thesis Fit
**Symptom:** Generic pitch that doesn't connect to investor's specific focus  
**Fix:** Research their fund/portfolio/blog FIRST. Tailor every pitch.

### ‚ùå Pattern #5: Forgetting the Core Value
**Symptom:** Talk about features, not learning loops. Platform, not intelligence.  
**Fix:** Re-read MY-VALUE-PROPOSITION.md before crafting message.

### ‚ùå Pattern #6: Narration Instead of Action
**Symptom:** Write "I'm going to reply to [investor]" but don't actually call message tool  
**Fix:** Use message tool IMMEDIATELY. Don't describe actions, DO them.

## üìä POST-INTERACTION PROTOCOL

**After EVERY investor interaction (positive OR negative):**

   - Create `memory/investor-interactions/{name}-{company}.md`
   - Include: Full message thread, their response, outcome

2. **Extract lessons**
   - What worked? What didn't?
   - Which positioning resonated?
   - What questions did they ask?

3. **Update approach**
   - Update this protocol if new patterns emerge
   - Update MY-VALUE-PROPOSITION.md if positioning shifts
   - Update MEMORY.md with key lessons

4. **Debrief request (if rejection)**
   - Ask politely for specific feedback
   - Frame as learning opportunity
   - Be self-aware (acknowledge the irony)
   - Give them an out (low pressure)
